# **사용자 흐름도 (User Flow): Sell-Master**

본 문서는 이커머스 통합 관리 SaaS 'Sell-Master'의 핵심 사용자 경험 경로를 정의합니다.

## **1\. 초기 온보딩 및 채널 연동 (Onboarding & Integration)**

셀러가 서비스를 시작하고 판매 채널(스마트스토어, 쿠팡 등)을 연결하는 과정입니다.

1. **회원가입/로그인:** 이메일 인증을 통한 가입 및 로그인.  
2. **연동 마켓 선택:** 연결하고자 하는 마켓 아이콘 클릭 (예: 쿠팡).  
3. **인증 정보 입력:** API Key 또는 판매자 계정 정보 입력.  
4. **연동 테스트:** 시스템이 마켓 API 호출을 통해 유효성 확인.  
5. **연동 완료:** '연동된 채널' 목록에 추가 및 초기 데이터 수집 시작.

## **2\. 상품 수집 및 통합 관리 (Product Management)**

여러 마켓에 흩어진 상품 정보를 하나로 모으고 편집하는 과정입니다.

1. **상품 가져오기:** 연동된 각 마켓에서 상품 정보 DB 수집.  
2. **마스터 상품 생성:** 서로 다른 채널의 동일 상품을 하나로 매핑(Binding).  
3. **일괄 수정:** 상품명, 판매가, 상세페이지 등을 시스템 내에서 수정.  
4. **채널 전송:** 수정사항을 선택한 모든 마켓에 한 번에 반영.  
5. **결과 확인:** 동기화 성공/실패 여부 대시보드 알림 확인.

## **3\. 주문 수집 및 배송 처리 (Order & Shipping)**

매일 발생하는 주문을 확인하고 송장을 입력하여 마켓에 전송하는 핵심 운영 과정입니다.

1. **통합 주문 조회:** '신규 주문' 탭에서 모든 마켓의 주문 확인.  
2. **주문 확정:** 발주 확인 처리를 진행하여 마켓에 '배송 준비 중' 상태 전송.  
3. **송장 번호 입력:** \* 방법 A: 엑셀 일괄 업로드.  
   * 방법 B: 택배사 API 연동 자동 생성.  
4. **배송 시작:** 송장 정보를 각 마켓 API로 전송.  
5. **상태 동기화:** 마켓에서 '배송 중'으로 변경된 것을 확인.

## **4\. 재고 관리 및 자동 품절 (Inventory Sync)**

판매에 따른 재고 차감과 품절 처리를 자동화하는 과정입니다.

1. **판매 발생:** 특정 채널(예: 쿠팡)에서 주문 발생.  
2. **재고 차감:** 시스템 내 '마스터 재고' 수량 자동 차감.  
3. **잔여 재고 계산:** 설정된 안전 재고 미만인지 확인.  
4. **채널 동기화:** \* 재고가 남은 경우: 다른 모든 채널(네이버 등)의 재고 수량 업데이트.  
   * 재고가 0인 경우: 모든 연동 채널에 '일시 품절' 상태 자동 전송.  
5. **알림 발송:** 셀러에게 '재고 부족' 또는 '품절 발생' 푸시 알림.

## **5\. 판매 분석 및 정산 확인 (Analytics)**

운영 성과를 확인하고 수익을 정산하는 과정입니다.

1. **통계 대시보드 접속:** 일별/주별/월별 전체 매출 그래프 확인.  
2. **채널별 비중 분석:** 어느 마켓의 매출 기여도가 높은지 파이 차트 확인.  
3. **정산 예정 조회:** 마켓별 정산 주기(구매확정 기준 등)에 따른 예상 입금액 확인.  
4. **리포트 다운로드:** 회무 처리를 위한 엑셀 데이터 추출.

## **6\. 예외 상황 처리 (Exception Handling)**

오류 발생 시 사용자가 대처하는 흐름입니다.

* **API 토큰 만료:** 연동 오류 알림 클릭 \-\> 마켓 설정 페이지 이동 \-\> 토큰 갱신(Re-auth).  
* **주문 취소 발생:** 마켓 웹훅 수신 \-\> 주문 리스트에서 '취소됨' 표시 \-\> 배송 중단 프로세스 안내.  
* **재고 불일치:** 마스터 상품 매핑 해제 및 재설정 기능을 통해 데이터 보정.
