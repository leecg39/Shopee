# **데이터베이스 설계서 (DB Schema): Sell-Master (Supabase 기반)**

본 문서는 Supabase(PostgreSQL) 환경에서 이커머스 통합 관리 SaaS를 구축하기 위한 데이터 모델링 및 보안 정책을 정의합니다.

## **1\. 개요**

* **Database:** PostgreSQL (Supabase Managed)  
* **Multi-tenancy:** workspace\_id를 기반으로 사용자/팀별 데이터 격리  
* **Security:** RLS(Row Level Security) 정책을 통해 인증된 사용자만 본인의 데이터에 접근

## **2\. 테이블 상세 설계**

### **2.1. 워크스페이스 및 권한 (Core)**

사용자와 데이터 사이의 소유권 단위를 정의합니다.

\-- 워크스페이스 (팀 단위)  
CREATE TABLE workspaces (  
  id UUID PRIMARY KEY DEFAULT gen\_random\_uuid(),  
  name TEXT NOT NULL,  
  owner\_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,  
  created\_at TIMESTAMPTZ DEFAULT NOW(),  
  updated\_at TIMESTAMPTZ DEFAULT NOW()  
);

\-- 워크스페이스 멤버십  
CREATE TABLE workspace\_members (  
  id UUID PRIMARY KEY DEFAULT gen\_random\_uuid(),  
  workspace\_id UUID REFERENCES workspaces(id) ON DELETE CASCADE,  
  user\_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,  
  role TEXT CHECK (role IN ('owner', 'admin', 'editor', 'viewer')),  
  UNIQUE(workspace\_id, user\_id)  
);

### **2.2. 판매 채널 연동 (Market Connections)**

각 마켓(쿠팡, 스마트스토어 등)의 API 인증 정보를 저장합니다.

CREATE TABLE market\_accounts (  
  id UUID PRIMARY KEY DEFAULT gen\_random\_uuid(),  
  workspace\_id UUID REFERENCES workspaces(id) ON DELETE CASCADE,  
  market\_type TEXT NOT NULL, \-- 'COUPANG', 'NAVER', '11ST' 등  
  account\_name TEXT NOT NULL, \-- 셀러 구분용 별칭  
  auth\_info JSONB NOT NULL, \-- API Key, Secret, Token 등 (암호화 권장)  
  sync\_status TEXT DEFAULT 'active', \-- 'active', 'expired', 'error'  
  last\_synced\_at TIMESTAMPTZ,  
  created\_at TIMESTAMPTZ DEFAULT NOW()  
);

### **2.3. 상품 관리 (Products)**

시스템 내 기준이 되는 '마스터 상품'과 실제 마켓에 등록된 '채널 상품'을 분리합니다.

\-- 마스터 상품 (기준 데이터)  
CREATE TABLE master\_products (  
  id UUID PRIMARY KEY DEFAULT gen\_random\_uuid(),  
  workspace\_id UUID REFERENCES workspaces(id) ON DELETE CASCADE,  
  sku\_code TEXT NOT NULL, \-- 자체 관리 코드  
  name TEXT NOT NULL,  
  base\_price NUMERIC NOT NULL DEFAULT 0,  
  stock\_quantity INTEGER NOT NULL DEFAULT 0,  
  category\_id TEXT,  
  metadata JSONB, \-- 추가 속성  
  created\_at TIMESTAMPTZ DEFAULT NOW(),  
  updated\_at TIMESTAMPTZ DEFAULT NOW()  
);

\-- 마켓별 연결된 상품 (Mapping)  
CREATE TABLE market\_products (  
  id UUID PRIMARY KEY DEFAULT gen\_random\_uuid(),  
  master\_product\_id UUID REFERENCES master\_products(id) ON DELETE SET NULL,  
  market\_account\_id UUID REFERENCES market\_accounts(id) ON DELETE CASCADE,  
  market\_product\_id\_external TEXT NOT NULL, \-- 마켓 측 상품 ID  
  market\_status TEXT, \-- '판매중', '품절', '중지'  
  sale\_price NUMERIC, \-- 마켓별 개별 가격  
  sync\_enabled BOOLEAN DEFAULT TRUE,  
  created\_at TIMESTAMPTZ DEFAULT NOW()  
);

### **2.4. 주문 및 재고 (Orders & Inventory)**

실시간 주문 수집 및 재고 변동 이력을 관리합니다.

\-- 통합 주문  
CREATE TABLE orders (  
  id UUID PRIMARY KEY DEFAULT gen\_random\_uuid(),  
  workspace\_id UUID REFERENCES workspaces(id) ON DELETE CASCADE,  
  market\_account\_id UUID REFERENCES market\_accounts(id),  
  external\_order\_id TEXT NOT NULL, \-- 마켓 주문번호  
  orderer\_name TEXT,  
  recipient\_name TEXT,  
  shipping\_address TEXT,  
  total\_amount NUMERIC,  
  order\_status TEXT, \-- 'pending', 'confirmed', 'shipping', 'completed', 'cancelled'  
  order\_date TIMESTAMPTZ NOT NULL,  
  created\_at TIMESTAMPTZ DEFAULT NOW()  
);

\-- 재고 변동 이력  
CREATE TABLE inventory\_logs (  
  id UUID PRIMARY KEY DEFAULT gen\_random\_uuid(),  
  master\_product\_id UUID REFERENCES master\_products(id) ON DELETE CASCADE,  
  change\_amount INTEGER NOT NULL,  
  reason TEXT, \-- 'order', 'adjustment', 'return'  
  reference\_id UUID, \-- 주문 ID 등 연관 데이터  
  created\_at TIMESTAMPTZ DEFAULT NOW()  
);

## **3\. RLS (Row Level Security) 정책**

Supabase 클라이언트에서 직접 접근할 때 데이터를 보호하기 위한 정책 예시입니다.

\-- RLS 활성화  
ALTER TABLE workspaces ENABLE ROW LEVEL SECURITY;  
ALTER TABLE market\_accounts ENABLE ROW LEVEL SECURITY;  
ALTER TABLE master\_products ENABLE ROW LEVEL SECURITY;

\-- 워크스페이스 접근 정책 (본인이 멤버인 경우만 조회 가능)  
CREATE POLICY "Users can view their own workspace members"  
ON workspace\_members FOR SELECT  
USING (auth.uid() \= user\_id);

\-- 상품 접근 정책 (워크스페이스 멤버십 확인)  
CREATE POLICY "Users can manage products in their workspace"  
ON master\_products  
FOR ALL  
USING (  
  workspace\_id IN (  
    SELECT workspace\_id FROM workspace\_members WHERE user\_id \= auth.uid()  
  )  
);

## **4\. 주요 인덱스 (Performance)**

* master\_products(sku\_code): 상품 검색 속도 향상  
* orders(external\_order\_id): 마켓 주문 중복 수집 방지  
* market\_products(market\_product\_id\_external): 마켓 상품 매핑 조회 최적화

## **5\. 설계 포인트**

1. **JSONB 활용:** 마켓마다 서로 다른 API 응답 스키마를 유연하게 수용하기 위해 metadata, auth\_info 필드에 JSONB 타입을 사용했습니다.  
2. **Master-Slave 구조:** master\_products가 재고와 기본 정보의 중심점이 되고, market\_products는 각 채널별로 변하는 정보(가격, 상태)를 담아 1:N 관계를 형성합니다.  
3. **확장성:** 새로운 마켓이 추가되어도 market\_accounts의 market\_type만 추가하면 되도록 설계되었습니다.
